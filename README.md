# recon
Why the name 'Recon'? I like the idea of searching for information in the field by means of detailed examination. That is the idea here. To examine and play with algorithms as a means to improve my skills as a developer. You will find varying degrees of investigation, and even some notes where I intend to come back to things. This is as much about what piques my curiosity as anything else. Some of these implementations also serve as reference examples when onboarding engineers or conducting code reviews, and I draw from this repo when forming technical interview questions; it helps me think clearly about what separates a working answer from a deep one. Also, please note that most often comments will be omitted...
